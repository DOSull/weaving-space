weavingspace.topology

Together the Topology, Tile, Vertex, Edge and weavingspace.symmetry. Symmetry and weavingspace.symmetry.Symmetries classes enable extraction of the topological structure of periodic weavingspace.tileable.Tileable objects so that modification of equivalent tiles can be carried out while retaining tileability. It is important to note that these are not fully generalised classes and methods, that is, the Topology object that is supported is not a permanent 'backing' data structure for our Tileable objects. While it might become that in time, as at Feb 2024 it is not such a data structure. Instead usage is

tile = TileUnit(...)
topology = Topology(tile)
topology = topology.transform_*(...)
new_tile = topology.tile_unit

Topology plot function is necessary for a user to be able to see what they are doing, because how edges and vertices in a tiling are labelled under tile equivalences is an essential step in the process.

Note also that these classes do not accurately represent the distinctions made in the mathematical literature between tiling vertices and tile corners, or between tiling edges and tile sides.

 127  def _initialise_points_into_tiles(self, debug:bool = False) -> None:
 128    """Sets up dictionary of unique point locations in the tiling and assigns
 129    them to Tiles.
 130    
 131    Args:
 132      debug (bool): if True prints useful debugging information.
 133    """
 134    shapes = self.tileable.get_local_patch(r = 1, include_0 = True).geometry
 135    shapes = [tiling_utils.get_clean_polygon(s) for s in shapes]
 136    self.n_1_order_patch = len(shapes)
 137    labels = list(self.tileable.tiles.tile_id) * (len(shapes) // self.n_tiles)
 138    self.tiles = []
 139    self.points = {}
 140    for (i, shape), label in zip(enumerate(shapes), labels):
 141      tile = Tile(i)
 142      tile.label = label
 143      tile.base_ID = tile.ID % self.n_tiles
 144      self.tiles.append(tile)
 145      tile.corners = []
 146      corners = tiling_utils.get_corners(shape, repeat_first = False)
 147      for c in corners:
 148        prev_vertex = None
 149        for p in reversed(self.points.values()):
 150          if c.distance(p.point) <= 2 * tiling_utils.RESOLUTION:
 151            # found an already existing vertex, so add to the tile and break
 152            prev_vertex = p
 153            tile.corners.append(prev_vertex)
 154            break
 155        if prev_vertex is None:
 156          # new vertex, add it to the topology dictionary and to the tile
 157          v = self.add_vertex(c)
 158          tile.corners.append(v)
 159          if debug: print(f"Added new Vertex {v} to Tile {i}")

 161  def _setup_vertex_tile_relations(self, debug:bool = False):
 162    """Determines relations between vertices and tiles. In particular vertices
 163    along tile edges that are not yet included in their list of vertices are
 164    added. Meanwhile vertex lists of incident tiles and neighbours are set up.
 165    
 166    Args:
 167      debug (bool): if True prints debugging information.
 168    """
 169    # we do this for all tiles in the radius-1 local patch
 170    for tile in self.tiles:
 171      if debug: print(f"Checking for vertices incident on Tile {tile.ID}")
 172      corners = []
 173      # performance is much improved using the vertex IDs
 174      initial_corner_IDs = tile.get_corner_IDs()
 175      # we need the current shape (not yet set) to check for incident vertices 
 176      shape = geom.Polygon([c.point for c in tile.corners])
 177      # get points incident on tile boundary, not already in the tile corners
 178      new_points = [v for v in self.points.values()
 179                    if v.ID not in initial_corner_IDs and
 180                    v.point.distance(shape) <= 2 * tiling_utils.RESOLUTION]
 181      # iterate over sides of the tile to see which the vertex is incident on
 182      for c1, c2 in zip(tile.corners, tile.corners[1:] + tile.corners[:1]):
 183        all_points = [c1, c2]
 184        if len(new_points) > 0:
 185          if debug: print(f"{[v.ID for v in new_points]} incident on tile")
 186          ls = geom.LineString([c1.point, c2.point])
 187          to_insert = [v for v in new_points 
 188                       if v.point.distance(ls) <= 2 * tiling_utils.RESOLUTION]
 189          if len(to_insert) > 0:
 190            # sort by distance along the side
 191            d_along = sorted([(ls.line_locate_point(v.point), v)
 192                              for v in to_insert], key = lambda x: x[0])
 193            to_insert = [v for d, v in d_along]
 194            all_points = all_points[:1] + to_insert + all_points[1:]
 195        corners.extend(all_points[:-1])
 196        for x1, x2 in zip(all_points[:-1], all_points[1:]):
 197          # x2 will add the tile and neigbour when we get to the next side
 198          # every vertex gets a turn!
 199          x1.add_tile(tile)
 200          x1.add_neighbour(x2.ID)
 201      tile.corners = corners
 202      tile.set_shape_from_corners()

 204  def _setup_edges(self, debug:bool = False):
 205    """Sets up the tiling edges. 
 206    
 207    First vertices in the base tiles are classified as tiling vertices or not -- 
 208    only these can be classified reliably (e.g vertices on the perimeter are 
 209    tricky).. Up to here all vertices have been considered tiling vertices.
 210    
 211    Second edges are created by traversing tile corner lists. Edges are stored
 212    once only by checking for edges in the reverse direction already in the 
 213    edges dictionary. Edge right and left tiles are initialised.
 214    
 215    Third tile edge direction lists are initialised.
 216    """
 217    # classify vertices in the base tiles
 218    for tile in self.tiles[:self.n_tiles]:
 219      for v in tile.corners:
 220        v.is_tiling_vertex = len(v.neighbours) > 2
 221    if debug: print(f"Classified base tile vertices")
 222    self.edges = {}
 223    for tile in self.tiles:
 224      if debug: print(f"Adding edges from Tile {tile.ID}")
 225      tile.edges = []
 226      vertices = [v for v in tile.corners if v.is_tiling_vertex]
 227      # note that through here finding ints in lists is much faster than
 228      # finding Vertex objects, hence we use lists of IDs not Vertex objects
 229      if len(vertices) > 1:
 230        for v1, v2 in zip(vertices, vertices[1:] + vertices[:1]):
 231          corner_IDs = tile.get_corner_IDs()
 232          idx1 = corner_IDs.index(v1.ID)
 233          idx2 = corner_IDs.index(v2.ID)
 234          if idx1 < idx2:
 235            corners = [c for c in corner_IDs[idx1:(idx2 + 1)]]
 236          else:
 237            corners = [c for c in corner_IDs[idx1:] + corner_IDs[:(idx2 + 1)]]
 238          ID = (corners[0], corners[-1])
 239          if not ID in self.edges:
 240            # check that reverse direction edge is not present first
 241            r_ID = ID[::-1]
 242            if r_ID in self.edges:
 243              # if it is, then add it and set left_tile
 244              if debug: print(f"reverse edge {r_ID} found")
 245              e = self.edges[r_ID]
 246              e.left_tile = tile
 247              tile.edges.append(e)
 248            else:
 249              # we've found a new edge so make and add it
 250              if debug: print(f"adding new edge {corners}")
 251              e = self.add_edge([self.points[c] for c in corners])
 252              e.right_tile = tile
 253              tile.edges.append(e)
 254      # initialise the edge direction information in the tile
 255      tile.set_edge_directions()

 267  def _assign_vertex_base_IDs(self, first_pass:bool = True,
 268                              last_pass:bool = False):
 269    """Assigns base_ID attribute of vertices. Two passes are required.
 270    
 271    TODO: consider if there is a networkx.connected_communities option
 272    here, since this is in essence about finding connected groups of tiles,
 273    vertices and edges. At very least, the need to run the code twice suggests
 274    that a more elegant approach is out there...
 275
 276    Args:
 277      first_pass (bool, optional): it True the core vertices on the tiles in 
 278        the Topology tileable are assigned; if False this step is skipped and
 279        IDs are propagated to neighbours. Defaults to True.
 280      last_pass (bool, optional): if True, it is the last pass and IDs are 
 281        finally assigned based on the lowest ID assigned in previous rounds. 
 282        Defaults to False.
 283    """
 284    if first_pass:
 285      for i, v in enumerate(self.vertices_in_tiles(self.tiles[:self.n_tiles])):
 286        v.base_ID = set([i])
 287    for tile in self.tiles:
 288      for v, vb in zip(tile.corners, self.tiles[tile.base_ID].corners):
 289        if v.base_ID == v.ID: # 1_000_000:
 290          v.base_ID = set([x for x in vb.base_ID])
 291        else:
 292          v.base_ID = v.base_ID.union(vb.base_ID)
 293    if last_pass:
 294      for tile in self.tiles:
 295        for v in tile.corners:
 296          if isinstance(v.base_ID, set):
 297            v.base_ID = min(v.base_ID)

 309  def _copy_base_tiles_to_patch(self):
 310    """Copies attributes of base tiles to their corresponding tiles in 
 311    the radius-1 patch. This requires:
 312     
 313    First, inserting any additional corners in the base tiles not found in the 
 314    radius-1 tiles. 
 315    
 316    Second, any vertices in the base tiles that are NOT tiling vertices are 
 317    applied to radius-1 tiles leading to merging of some edges.
 318    """
 319    # the number of tiles in the base + radius-1
 320    n_r1 = len(self.tiles)
 321    # first add any missing vertices to the non-base tiles
 322    # we add all the missing vertices before doing any merges
 323    for base in self.tiles[:self.n_tiles]:
 324      for other in self.tiles[base.ID:n_r1:self.n_tiles]:
 325        self._match_reference_tile_vertices(base, other)
 326    # then merge any edges that meet at a corner
 327    for base in self.tiles[:self.n_tiles]:
 328      for other in self.tiles[base.ID:n_r1:self.n_tiles]:
 329        self._match_reference_tile_corners(base, other)

 331  def _match_reference_tile_vertices(self, tile1:Tile, tile2:Tile):
 332    """Adds vertices to tile2 so that it matches tile1 adjusting edges as
 333    required. This assumes the tiles are the same shape, but that tile2 may
 334    be missing some tiling vertices along some edges.
 335
 336    Args:
 337      tile1 (Tile): reference tile.
 338      tile2 (Tile): tile to change.
 339    """
 340    to_add = len(tile1.corners) - len(tile2.corners)
 341    while to_add > 0:
 342      # find the reference x-y offset 
 343      dxy = (tile2.centre.x - tile1.centre.x, tile2.centre.y - tile1.centre.y)
 344      for i, t1c in enumerate([c.point for c in tile1.corners]):
 345        t2c = tile2.corners[i % len(tile2.corners)].point
 346        if abs((t2c.x - t1c.x) - dxy[0]) > 10 * tiling_utils.RESOLUTION or \
 347           abs((t2c.y - t1c.y) - dxy[1]) > 10 * tiling_utils.RESOLUTION:
 348          # add vertex to t2 by copying the t1 vertex appropriately offset
 349          # note that this might alter the length of t2.corners
 350          v = self.add_vertex(geom.Point(t1c.x + dxy[0], t1c.y + dxy[1]))
 351          v.is_tiling_vertex = True
 352          old_edge, new_edges = tile2.insert_vertex_at(v, i)
 353          del self.edges[old_edge]
 354          for e in new_edges:
 355            self.edges[e.ID] = e
 356          to_add = to_add - 1

 358  def _match_reference_tile_corners(self, tile1:Tile, tile2:Tile):
 359    """Finds vertices that are corners in tile2 but vertices in tile1 and 
 360    updates tile2 to match - merging edges as required.
 361
 362    Args:
 363        tile1 (Tile): reference tile.
 364        tile2 (Tile): tile to make match.
 365    """
 366    vs_to_change = [vj for vi, vj in zip(tile1.corners, tile2.corners)
 367                    if not vi.is_tiling_vertex and vj.is_tiling_vertex]
 368    if len(vs_to_change) > 0:
 369      for v in vs_to_change:
 370        v.is_tiling_vertex = False
 371        # it's a corner not an edge so will have no more than 2 v.tiles
 372        old_edges, new_edge = v.tiles[0].merge_edges_at_vertex(v)
 373        for e in old_edges:
 374          del self.edges[e]
 375        self.edges[new_edge.ID] = new_edge

 377  def _identify_distinct_tile_shapes(self, ignore_tile_id_labels:bool = True):
 378    """Determines unique tiles based on their symmetries and shapes. At the
 379    same time assembles a list of the affine transforms under which matches
 380    occurs since these are potential symmetries of the tiling.
 381    
 382    TODO: reimplement consideration of tile_id
 383    
 384    Args:
 385      ignore_tile_id_labels (bool): if True only the shape of tiles matters; if 
 386        False the tile_id label is also considered. Defaults to True.
 387    """
 388    matches = {}
 389    offsets = {}
 390    for tile in self.tiles[:self.n_tiles]:
 391      matches[tile.base_ID] = [tile.base_ID]
 392      matched = False
 393      s = Symmetries(tile.shape)
 394      for other in self.tiles[:self.n_tiles]:
 395        if other.ID > tile.ID:
 396          offset = s.get_corner_offset(other.shape)
 397          if not offset is None:
 398            offsets[tile.base_ID] = offset
 399            matches[tile.base_ID].append(other.base_ID)
 400            matched = True
 401      if not matched:
 402        offsets[tile.base_ID] = 0
 403    base_groups = list(nx.connected_components(nx.from_dict_of_lists(matches)))
 404    self.shape_groups = []
 405    for i, group in enumerate(base_groups):
 406      full_group = []
 407      for tile in self.tiles:
 408        if tile.base_ID in group:
 409          tile.shape_group = i
 410          tile.offset_corners(offsets[tile.base_ID])
 411          full_group.append(tile.ID)
 412      self.shape_groups.append(full_group)

 414  def _find_tile_transitivity_classes(self):
 415    """Finds tiles equivalent under symmetries, at the same time updating the 
 416    tile_matching_transforms attribute to contain only those transforms that 
 417    pass this test. 
 418    """
 419    self.tile_matching_transforms = self.get_potential_symmetries()
 420    base_tiles = [t for t in self.tiles[:self.n_tiles]]
 421    # it is quicker (should be!) to only do within shape group tests
 422    # often there is only one when it will make no difference
 423    by_group_equivalent_tiles = []
 424    # maintain a set of transforms still potentially tiling symmetries
 425    poss_transforms = set(self.tile_matching_transforms.keys())
 426    # and a dictionary of booleans tracking which transforms are still valid
 427    eq_under_transform = {tr: True for tr in poss_transforms}
 428    for g, group in enumerate(self.shape_groups):
 429      by_group_equivalent_tiles.append(set())
 430      source_tiles = [tile for tile in base_tiles if tile.shape_group == g]
 431      target_tiles = [tile for tile in self.tiles if tile.shape_group == g]
 432      for tr in poss_transforms:
 433        transform = self.tile_matching_transforms[tr].transform
 434        matched_tiles = {}
 435        eq_under_transform[tr] = True
 436        for source_tile in source_tiles:
 437          matched_tile_id = self._match_geoms_under_transform(
 438            source_tile, target_tiles, transform)
 439          if matched_tile_id == -1:
 440            eq_under_transform[tr] = False
 441            break
 442          else:
 443            matched_tiles[source_tile.ID] = matched_tile_id
 444        if eq_under_transform[tr]:
 445          for k, v in matched_tiles.items():
 446            # print(f"src {k} tgt {v} {tr=} {transform}")
 447            # here we record the transform, in case it is later invalidated
 448            by_group_equivalent_tiles[g].add((tr, k, v))
 449      # remove valid transforms that didn't make it through this group
 450      poss_transforms = set([t for t, x in eq_under_transform.items() if x])
 451    # compile equivalences from all groups made under still valid transforms
 452    # a dict of sets so singletons aren't lost in finding connected components
 453    equivalents = {i: set() for i in range(self.n_tiles)}
 454    for group_equivalents in by_group_equivalent_tiles:
 455      for (tr, tile_i, tile_j) in group_equivalents:
 456        if tr in poss_transforms:
 457          equivalents[tile_i].add(tile_j)
 458    self.tile_matching_transforms = {
 459      k: v for k, v in self.tile_matching_transforms.items() if k in 
 460      poss_transforms}
 461    self.tile_transitivity_classes = []
 462    equivalents = nx.connected_components(nx.from_dict_of_lists(equivalents))
 463    for c, base_IDs in enumerate(equivalents):
 464      transitivity_class = []
 465      for tile in self.tiles:
 466        if tile.base_ID in base_IDs:
 467          transitivity_class.append(tile.ID)
 468          tile.transitivity_class = c
 469      self.tile_transitivity_classes.append(transitivity_class)

 471  def get_potential_symmetries(self) -> dict[int, tuple[float]]:
 472    """Assembles potential symmetries of the tiling from symmetries of the 
 473    tileable.prototile and of the tileable.tiles. Removes any duplicates that
 474    result. Result is assigned to the tile_matching_transforms attribute.
 475    
 476    TODO: consider retaining the Symmetry objects as these carry additional 
 477    information that might facilitate labelling under a limited number of the
 478    symmetries not all of them.
 479
 480    Returns:
 481      dict[int, tuple[float]]: dictionary of the symmetries (transforms 
 482        actually) in shapely affine transform 6-tuple format.
 483    """
 484    self.tile_matching_transforms = {}
 485    n_symmetries = 0
 486    pt = self.tileable.prototile.geometry[0]
 487    for tr in Shape_Matcher(pt).get_polygon_matches(pt):
 488      if not tr.transform_type in ["identity", "translation"]:
 489        self.tile_matching_transforms[n_symmetries] = tr
 490        n_symmetries = n_symmetries + 1
 491    for tile in self.tiles[:self.n_tiles]:
 492      for tr in Shape_Matcher(tile.shape).get_polygon_matches(tile.shape):
 493        if not tr.transform_type in ["identity", "translation"]:
 494          self.tile_matching_transforms[n_symmetries] = tr
 495          n_symmetries = n_symmetries + 1
 496    for tile in self.tiles[:self.n_tiles]:
 497      sm = Shape_Matcher(tile.shape)
 498      transforms = [sm.get_polygon_matches(self.tiles[i].shape) 
 499        for i in self.shape_groups[tile.shape_group] if i < self.n_tiles]
 500      for tr in itertools.chain(*transforms):
 501        if not tr.transform_type in ["identity", "translation"]:
 502          self.tile_matching_transforms[n_symmetries] = tr
 503          n_symmetries = n_symmetries + 1
 504    self.tile_matching_transforms = self._remove_duplicate_symmetries(
 505      self.tile_matching_transforms)
 506    return self.tile_matching_transforms

 508  def _remove_duplicate_symmetries(self, transforms:dict[int,Any]):
 509    """Filters the supplied list of shapely affine transforms so that no 
 510    duplicates are retained.
 511    """
 512    uniques = {}
 513    for k, v in transforms.items():
 514      already_exists = False
 515      for i, u in uniques.items():
 516        already_exists = np.allclose(
 517          v.transform, u.transform, atol = 1e-4, rtol = 1e-4)
 518        if already_exists:
 519          break
 520      if not already_exists:
 521        uniques[k] = v
 522    return uniques

 591  def _match_geoms_under_transform(
 592      self, geom1:Union[Tile,Vertex,Edge], 
 593      geoms2:list[Union[Tile,Vertex,Edge]], transform:tuple[float]) -> int:
 594    """Determines if there is a geometry in the supplied patch onto which the
 595    supplied geometry is mapped by the supplied symmetry.
 596
 597    Args:
 598      tile (Union[Tile,Vertex,Edge]): element whose geometry we want to match.
 599      patch (list[Union[Tile,Vertex,Edge]]): set of elements among which a 
 600        match is sought.
 601      transform (tuple[float]): shapely affine transform 6-tuple to apply.
 602
 603    Returns:
 604      Union[int,tuple[int]]: ID of the element in patch that matches the geom1
 605        element under the transform if one exists, otherwise returns -1.
 606    """
 607    match_id = -1
 608    for geom2 in geoms2:
 609      if isinstance(geom1, Tile):
 610        # an area of intersection based test
 611        match = tiling_utils.geometry_matches(
 612          affine.affine_transform(geom1.shape, transform), geom2.shape)
 613      elif isinstance(geom1, Vertex):
 614        # distance test
 615        match = affine.affine_transform(geom1.point, transform).distance(
 616          geom2.point) <= 10 * tiling_utils.RESOLUTION
 617      else: # must be an Edge
 618        # since edges _should not_ intersect this test should work in
 619        # lieu of a more complete point by point comparison
 620        c1 = geom1.get_geometry().centroid
 621        c2 = geom2.get_geometry().centroid
 622        match = affine.affine_transform(c1, transform) \
 623          .distance(c2) <= 10 *tiling_utils.RESOLUTION
 624      if match:
 625        return geom2.base_ID
 626    return match_id

 628  def _get_exclusive_supersets(
 629      self, sets:list[Iterable[Any]]) -> list[Iterable[Any]]:
 630    """For the supplied list of lists, which may share elements, i.e. they are
 631    non-exclusives sets, returns a list of lists which are exclusive: each 
 632    element will only appear in one of the lists in the returned list. Uses
 633    networkx connected components function to achieve this based on a graph 
 634    where an intersection between two sets is an edge.
 635
 636    Args:
 637        sets (list[Iterable[Any]]): list of lists of possibly overlapping sets.
 638
 639    Returns:
 640      list[Iterable[Any]]: list of lists that include all the original 
 641        elements without overlaps.
 642    """
 643    overlaps = []
 644    for i, si in enumerate(sets):
 645      s1 = set(si)
 646      for j, sj in enumerate(sets):
 647        s2 = set(sj)
 648        if len(s1 & s2) > 0:
 649          overlaps.append((i, j))
 650    G = nx.from_edgelist(overlaps)
 651    result = []
 652    for component in nx.connected_components(G):
 653      s = set()
 654      for i in component:
 655        s = s.union(sets[i])
 656      result.append(tuple(s))
 657    return result

 834  def transform_geometry(self, new_topology:bool, apply_to_tiles:bool,
 835                         selector:str, type:str, **kwargs) -> "Topology":
 836    r"""Applies a specified transformation of elements in the Topology whose 
 837    labels match the selector parameter, optionally applying the transform to 
 838    update tile and optionally returning a new Topology object (or applying it 
 839    to this one). 
 840    
 841    Implemented in this way so that transformations can be applied one at a time
 842    without creating an intermediate set of new tiles, which may be invalid and
 843    fail. So, if you wish to apply (say) 3 transforms and generate a new 
 844    Topology leaving the existing one intact:
 845    
 846        new_topo = old_topo.transform_geometry(True,  False, "a", ...) \
 847                           .transform_geometry(False, False, "B", ...) \
 848                           .transform_geometry(False, True,  "C", ...)
 849    
 850    The first transform requests a new Topology, subsequent steps do not, and it
 851    is only the last step which attempts to create the new tile polygons.
 852    
 853    **kwargs supply named parameters for the requested transformation.
 854
 855    Args:
 856      new_topology (bool): if True returns a new Topology object, else returns 
 857        the current Topology modified.
 858      apply_to_tiles (bool): if True attempts to create new Tiles after the 
 859        transformation has been applied. Usually set to False, unless the last
 860        transformation in a pipeline, to avoid problems of topologically invalid
 861        tiles at intermediate steps.
 862      selector (str): label of elements to which to apply the transformation.   
 863        Note that all letters in the supplied string are checked, so you can 
 864        use e.g. "abc" to apply a transformation to edges labelled "a", "b" or 
 865        "c", or "AB" for vertices labelled "A" or "B".
 866      type (str): name of the type of transformation requested. Currently
 867        supported are `zigzag_edge`, `rotate_edge`, `push_vertex`, and 
 868        `nudge_vertex`. Keyword arguments for each are documented in the 
 869        corresponding methods.
 870
 871    Returns:
 872      Topology: if new_topology is True a new Topology based on this one with
 873        after transformation, if False this Topology is returned after the
 874        transformation.
 875    """
 876    print(
 877f"""CAUTION: new Topology will probably not be correctly labelled. To build a 
 878correct Topology, extract the tileable attribute and rebuild Topology from that.
 879""")
 880    topo = pickle.loads(pickle.dumps(self)) if new_topology else self
 881    transform_args = topo.get_kwargs(getattr(topo, type), **kwargs)
 882    if type == "zigzag_edge":
 883      for e in topo.edges.values():
 884        if e.label in selector:
 885          topo.zigzag_edge(e, **transform_args)
 886    # ---------------------------
 887    elif type == "rotate_edge":
 888      for e in topo.edges.values():
 889        if e.label in selector:
 890          topo.rotate_edge(e, **transform_args)
 891    # ---------------------------
 892    elif type == "push_vertex":
 893      pushes = {}
 894      for v in topo.vertices_in_tiles(topo.tiles[:topo.n_tiles]):
 895        if v.label in selector:
 896          pushes[v.base_ID] = topo.push_vertex(v, **transform_args)
 897      for base_ID, (dx, dy) in pushes.items():
 898        for v in [v for v in topo.points.values() if v.base_ID == base_ID]:
 899          v.point = affine.translate(v.point, dx, dy)
 900    # ---------------------------
 901    elif type == "nudge_vertex":
 902       for v in topo.points.values():
 903         if v.label in selector:
 904           topo.nudge_vertex(v, **transform_args)
 905    if apply_to_tiles:
 906      for t in topo.tiles:
 907        t.set_corners_from_edges()
 908    topo.tileable.tiles.geometry = gpd.GeoSeries(
 909      [topo.tiles[i].shape for i in range(topo.n_tiles)])
 910    topo.tileable.setup_regularised_prototile_from_tiles()
 911    return topo

 913  def zigzag_edge(self, edge:Edge, start:str = "A",
 914                  n:int = 2, h:float = 0.5, smoothness:int = 0):
 915    """Applies a zigzag transformation to the supplied Edge. Currently this will
 916    only work correctly if h is even.
 917    
 918    TODO: make it possible for odd numbers of 'peaks' to work (this may require
 919    allowing bidirectional Edges, i.e. storing Edges in both directions so that
 920    all Tile edges are drawn CW). The `start` parameter is a temporary hack for 
 921    this
 922
 923    Args:
 924      edge (Edge): Edge to transform
 925      n (int, optional): number of zigs and zags in the edge. Defaults to 2.
 926      start (str, optional): label at one end of edge which is used to determine
 927        the sense of h, enabling C-curves with an odd number n of zigs and zags 
 928        to be applied. Defaults to 'A'.
 929      h (float, optional): width of the zig zags relative to edge length. 
 930        Defaults to 0.5.
 931      smoothness (int, optional): spline smoothness. 0 gives a zig zag proper,
 932        higher values will produce a sinusoid. Defaults to 0.
 933    """
 934    v0, v1 = edge.vertices[0], edge.vertices[1]
 935    if n % 2 == 1 and v0.label != start:
 936      h = -h
 937    ls = tiling_utils.zigzag_between_points(v0.point, v1.point, 
 938                                            n, h, smoothness)
 939    # remove current corners
 940    self.points = {k: v for k, v in self.points.items()
 941                   if not k in edge.get_corner_IDs()[1:-1]}
 942    # add the new ones
 943    new_corners = [self.add_vertex(geom.Point(xy)) for xy in ls.coords]
 944    edge.corners = edge.vertices[:1] + new_corners + edge.vertices[-1:]
 945    if not edge.right_tile is None:
 946      edge.right_tile.set_corners_from_edges(False)
 947    if not edge.left_tile is None:
 948      edge.left_tile.set_corners_from_edges(False)

 950  def rotate_edge(self, edge:Edge, centre:str = "A", angle:float = 0) -> None:
 951    v0, v1 = edge.vertices[0], edge.vertices[1]
 952    if v0.label == centre:
 953      c = v0.point
 954    elif v1.label == centre:
 955      c = v1.point
 956    else:
 957      c = ls.centroid
 958    ls = affine.rotate(geom.LineString([v0.point, v1.point]), angle, origin = c)
 959    v0.point, v1.point = [geom.Point(c) for c in ls.coords]

 961  def push_vertex(self, vertex:Vertex, push_d:float) -> tuple[float]:
 962    neighbours = [self.points[v] for v in vertex.neighbours]
 963    dists = [vertex.point.distance(v.point) for v in neighbours]
 964    x, y = vertex.point.x, vertex.point.y
 965    unit_vectors = [((x - v.point.x) / d, (y - v.point.y) / d)
 966                    for v, d in zip(neighbours, dists)]
 967    return  (push_d * sum([xy[0] for xy in unit_vectors]),
 968             push_d * sum([xy[1] for xy in unit_vectors]))

1006    from AB -> a+ BC -> b+ CB -> b- BA -> a- AA -> c, where + designated CW
1007    traversal, - CCW, and no symbol means that edge appears only once.
1008    
1009    This may yet have use in more elaborate edge transformations. (See G&S 87 
1010    on tile incidence symbols.)
1011
1012    Args:
1013      vlabels (list[str]): ordered list of vertex labels.
1014
1015    Returns:
1016      list[str]: corresponding list of edge labels.
1017    """
1018    AB_labels = [a + b for a, b in zip(vlabels, vlabels[1:] + vlabels[:1])]
1019    letter = ALPHABET.index(min(list(vlabels)))
1020    edge_labels = {}
1021    for AB_label in AB_labels:
1022      if not AB_label in edge_labels:
1023        if AB_label[::-1] in edge_labels:
1024          label = edge_labels[AB_label[::-1]]
1025          edge_labels[AB_label] = label + "-"
1026          edge_labels[AB_label[::-1]] = label + "+"
1027        else:
1028          edge_labels[AB_label] = alphabet[letter]
1029          letter = letter + 1
1030    return [edge_labels[i] for i in AB_labels]

1094  def _cyclic_sort_first(self, lst:Iterable) -> Iterable:
1095    """Returns supplied Iterable in canonical cyclic sorted form. E.g. the 
1096    sequence ACABD, yields 5 possible cycles ACABD, CABDA, ABDAC, BDACA, and 
1097    DACAB. The lexically first of these is ABDAC, which would be returned. 
1098
1099    Args:
1100        lst (Iterable): an Iterable of elements (usu. strings).
1101
1102    Returns:
1103      Iterable: the lexically first of the possible cycles in the supplied 
1104        iterable.
1105    """
1106    cycle = lst + lst
1107    n = len(lst)
1108    cycles = [cycle[i:i + n] for i in range(n)]
1109    return sorted(cycles)[0]

 128  def _initialise_points_into_tiles(self, debug:bool = False) -> None:
 129    """Sets up dictionary of unique point locations in the tiling and assigns
 130    them to Tiles.
 131    
 132    Args:
 133      debug (bool): if True prints useful debugging information.
 134    """
 135    shapes = self.tileable.get_local_patch(r = 1, include_0 = True).geometry
 136    shapes = [tiling_utils.get_clean_polygon(s) for s in shapes]
 137    self.n_1_order_patch = len(shapes)
 138    labels = list(self.tileable.tiles.tile_id) * (len(shapes) // self.n_tiles)
 139    self.tiles = []
 140    self.points = {}
 141    for (i, shape), label in zip(enumerate(shapes), labels):
 142      tile = Tile(i)
 143      tile.label = label
 144      tile.base_ID = tile.ID % self.n_tiles
 145      self.tiles.append(tile)
 146      tile.corners = []
 147      corners = tiling_utils.get_corners(shape, repeat_first = False)
 148      for c in corners:
 149        prev_vertex = None
 150        for p in reversed(self.points.values()):
 151          if c.distance(p.point) <= 2 * tiling_utils.RESOLUTION:
 152            # found an already existing vertex, so add to the tile and break
 153            prev_vertex = p
 154            tile.corners.append(prev_vertex)
 155            break
 156        if prev_vertex is None:
 157          # new vertex, add it to the topology dictionary and to the tile
 158          v = self.add_vertex(c)
 159          tile.corners.append(v)
 160          if debug: print(f"Added new Vertex {v} to Tile {i}")

 162  def _setup_vertex_tile_relations(self, debug:bool = False):
 163    """Determines relations between vertices and tiles. In particular vertices
 164    along tile edges that are not yet included in their list of vertices are
 165    added. Meanwhile vertex lists of incident tiles and neighbours are set up.
 166    
 167    Args:
 168      debug (bool): if True prints debugging information.
 169    """
 170    # we do this for all tiles in the radius-1 local patch
 171    for tile in self.tiles:
 172      if debug: print(f"Checking for vertices incident on Tile {tile.ID}")
 173      corners = []
 174      # performance is much improved using the vertex IDs
 175      initial_corner_IDs = tile.get_corner_IDs()
 176      # we need the current shape (not yet set) to check for incident vertices 
 177      shape = geom.Polygon([c.point for c in tile.corners])
 178      # get points incident on tile boundary, not already in the tile corners
 179      new_points = [v for v in self.points.values()
 180                    if v.ID not in initial_corner_IDs and
 181                    v.point.distance(shape) <= 2 * tiling_utils.RESOLUTION]
 182      # iterate over sides of the tile to see which the vertex is incident on
 183      for c1, c2 in zip(tile.corners, tile.corners[1:] + tile.corners[:1]):
 184        all_points = [c1, c2]
 185        if len(new_points) > 0:
 186          if debug: print(f"{[v.ID for v in new_points]} incident on tile")
 187          ls = geom.LineString([c1.point, c2.point])
 188          to_insert = [v for v in new_points 
 189                       if v.point.distance(ls) <= 2 * tiling_utils.RESOLUTION]
 190          if len(to_insert) > 0:
 191            # sort by distance along the side
 192            d_along = sorted([(ls.line_locate_point(v.point), v)
 193                              for v in to_insert], key = lambda x: x[0])
 194            to_insert = [v for d, v in d_along]
 195            all_points = all_points[:1] + to_insert + all_points[1:]
 196        corners.extend(all_points[:-1])
 197        for x1, x2 in zip(all_points[:-1], all_points[1:]):
 198          # x2 will add the tile and neigbour when we get to the next side
 199          # every vertex gets a turn!
 200          x1.add_tile(tile)
 201          x1.add_neighbour(x2.ID)
 202      tile.corners = corners
 203      tile.set_shape_from_corners()

 205  def _setup_edges(self, debug:bool = False):
 206    """Sets up the tiling edges. 
 207    
 208    First vertices in the base tiles are classified as tiling vertices or not -- 
 209    only these can be classified reliably (e.g vertices on the perimeter are 
 210    tricky).. Up to here all vertices have been considered tiling vertices.
 211    
 212    Second edges are created by traversing tile corner lists. Edges are stored
 213    once only by checking for edges in the reverse direction already in the 
 214    edges dictionary. Edge right and left tiles are initialised.
 215    
 216    Third tile edge direction lists are initialised.
 217    """
 218    # classify vertices in the base tiles
 219    for tile in self.tiles[:self.n_tiles]:
 220      for v in tile.corners:
 221        v.is_tiling_vertex = len(v.neighbours) > 2
 222    if debug: print(f"Classified base tile vertices")
 223    self.edges = {}
 224    for tile in self.tiles:
 225      if debug: print(f"Adding edges from Tile {tile.ID}")
 226      tile.edges = []
 227      vertices = [v for v in tile.corners if v.is_tiling_vertex]
 228      # note that through here finding ints in lists is much faster than
 229      # finding Vertex objects, hence we use lists of IDs not Vertex objects
 230      if len(vertices) > 1:
 231        for v1, v2 in zip(vertices, vertices[1:] + vertices[:1]):
 232          corner_IDs = tile.get_corner_IDs()
 233          idx1 = corner_IDs.index(v1.ID)
 234          idx2 = corner_IDs.index(v2.ID)
 235          if idx1 < idx2:
 236            corners = [c for c in corner_IDs[idx1:(idx2 + 1)]]
 237          else:
 238            corners = [c for c in corner_IDs[idx1:] + corner_IDs[:(idx2 + 1)]]
 239          ID = (corners[0], corners[-1])
 240          if not ID in self.edges:
 241            # check that reverse direction edge is not present first
 242            r_ID = ID[::-1]
 243            if r_ID in self.edges:
 244              # if it is, then add it and set left_tile
 245              if debug: print(f"reverse edge {r_ID} found")
 246              e = self.edges[r_ID]
 247              e.left_tile = tile
 248              tile.edges.append(e)
 249            else:
 250              # we've found a new edge so make and add it
 251              if debug: print(f"adding new edge {corners}")
 252              e = self.add_edge([self.points[c] for c in corners])
 253              e.right_tile = tile
 254              tile.edges.append(e)
 255      # initialise the edge direction information in the tile
 256      tile.set_edge_directions()

 310  def _copy_base_tiles_to_patch(self):
 311    """Copies attributes of base tiles to their corresponding tiles in 
 312    the radius-1 patch. This requires:
 313     
 314    First, inserting any additional corners in the base tiles not found in the 
 315    radius-1 tiles. 
 316    
 317    Second, any vertices in the base tiles that are NOT tiling vertices are 
 318    applied to radius-1 tiles leading to merging of some edges.
 319    """
 320    # the number of tiles in the base + radius-1
 321    n_r1 = len(self.tiles)
 322    # first add any missing vertices to the non-base tiles
 323    # we add all the missing vertices before doing any merges
 324    for base in self.tiles[:self.n_tiles]:
 325      for other in self.tiles[base.ID:n_r1:self.n_tiles]:
 326        self._match_reference_tile_vertices(base, other)
 327    # then merge any edges that meet at a corner
 328    for base in self.tiles[:self.n_tiles]:
 329      for other in self.tiles[base.ID:n_r1:self.n_tiles]:
 330        self._match_reference_tile_corners(base, other)

 332  def _match_reference_tile_vertices(self, tile1:Tile, tile2:Tile):
 333    """Adds vertices to tile2 so that it matches tile1 adjusting edges as
 334    required. This assumes the tiles are the same shape, but that tile2 may
 335    be missing some tiling vertices along some edges.
 336
 337    Args:
 338      tile1 (Tile): reference tile.
 339      tile2 (Tile): tile to change.
 340    """
 341    to_add = len(tile1.corners) - len(tile2.corners)
 342    while to_add > 0:
 343      # find the reference x-y offset 
 344      dxy = (tile2.centre.x - tile1.centre.x, tile2.centre.y - tile1.centre.y)
 345      for i, t1c in enumerate([c.point for c in tile1.corners]):
 346        t2c = tile2.corners[i % len(tile2.corners)].point
 347        if abs((t2c.x - t1c.x) - dxy[0]) > 10 * tiling_utils.RESOLUTION or \
 348           abs((t2c.y - t1c.y) - dxy[1]) > 10 * tiling_utils.RESOLUTION:
 349          # add vertex to t2 by copying the t1 vertex appropriately offset
 350          # note that this might alter the length of t2.corners
 351          v = self.add_vertex(geom.Point(t1c.x + dxy[0], t1c.y + dxy[1]))
 352          v.is_tiling_vertex = True
 353          old_edge, new_edges = tile2.insert_vertex_at(v, i)
 354          del self.edges[old_edge]
 355          for e in new_edges:
 356            self.edges[e.ID] = e
 357          to_add = to_add - 1

 359  def _match_reference_tile_corners(self, tile1:Tile, tile2:Tile):
 360    """Finds vertices that are corners in tile2 but vertices in tile1 and 
 361    updates tile2 to match - merging edges as required.
 362
 363    Args:
 364        tile1 (Tile): reference tile.
 365        tile2 (Tile): tile to make match.
 366    """
 367    vs_to_change = [vj for vi, vj in zip(tile1.corners, tile2.corners)
 368                    if not vi.is_tiling_vertex and vj.is_tiling_vertex]
 369    if len(vs_to_change) > 0:
 370      for v in vs_to_change:
 371        v.is_tiling_vertex = False
 372        # it's a corner not an edge so will have no more than 2 v.tiles
 373        old_edges, new_edge = v.tiles[0].merge_edges_at_vertex(v)
 374        for e in old_edges:
 375          del self.edges[e]
 376        self.edges[new_edge.ID] = new_edge

 378  def _identify_distinct_tile_shapes(self, ignore_tile_id_labels:bool = True):
 379    """Determines unique tiles based on their symmetries and shapes. At the
 380    same time assembles a list of the affine transforms under which matches
 381    occurs since these are potential symmetries of the tiling.
 382    
 383    TODO: reimplement consideration of tile_id
 384    
 385    Args:
 386      ignore_tile_id_labels (bool): if True only the shape of tiles matters; if 
 387        False the tile_id label is also considered. Defaults to True.
 388    """
 389    matches = {}
 390    offsets = {}
 391    for tile in self.tiles[:self.n_tiles]:
 392      matches[tile.base_ID] = [tile.base_ID]
 393      matched = False
 394      s = Symmetries(tile.shape)
 395      for other in self.tiles[:self.n_tiles]:
 396        if other.ID > tile.ID:
 397          offset = s.get_corner_offset(other.shape)
 398          if not offset is None:
 399            offsets[tile.base_ID] = offset
 400            matches[tile.base_ID].append(other.base_ID)
 401            matched = True
 402      if not matched:
 403        offsets[tile.base_ID] = 0
 404    base_groups = list(nx.connected_components(nx.from_dict_of_lists(matches)))
 405    self.shape_groups = []
 406    for i, group in enumerate(base_groups):
 407      full_group = []
 408      for tile in self.tiles:
 409        if tile.base_ID in group:
 410          tile.shape_group = i
 411          tile.offset_corners(offsets[tile.base_ID])
 412          full_group.append(tile.ID)
 413      self.shape_groups.append(full_group)

 415  def _find_tile_transitivity_classes(self):
 416    """Finds tiles equivalent under symmetries, at the same time updating the 
 417    tile_matching_transforms attribute to contain only those transforms that 
 418    pass this test. 
 419    """
 420    self.tile_matching_transforms = self.get_potential_symmetries()
 421    base_tiles = [t for t in self.tiles[:self.n_tiles]]
 422    # it is quicker (should be!) to only do within shape group tests
 423    # often there is only one when it will make no difference
 424    by_group_equivalent_tiles = []
 425    # maintain a set of transforms still potentially tiling symmetries
 426    poss_transforms = set(self.tile_matching_transforms.keys())
 427    # and a dictionary of booleans tracking which transforms are still valid
 428    eq_under_transform = {tr: True for tr in poss_transforms}
 429    for g, group in enumerate(self.shape_groups):
 430      by_group_equivalent_tiles.append(set())
 431      source_tiles = [tile for tile in base_tiles if tile.shape_group == g]
 432      target_tiles = [tile for tile in self.tiles if tile.shape_group == g]
 433      for tr in poss_transforms:
 434        transform = self.tile_matching_transforms[tr].transform
 435        matched_tiles = {}
 436        eq_under_transform[tr] = True
 437        for source_tile in source_tiles:
 438          matched_tile_id = self._match_geoms_under_transform(
 439            source_tile, target_tiles, transform)
 440          if matched_tile_id == -1:
 441            eq_under_transform[tr] = False
 442            break
 443          else:
 444            matched_tiles[source_tile.ID] = matched_tile_id
 445        if eq_under_transform[tr]:
 446          for k, v in matched_tiles.items():
 447            # print(f"src {k} tgt {v} {tr=} {transform}")
 448            # here we record the transform, in case it is later invalidated
 449            by_group_equivalent_tiles[g].add((tr, k, v))
 450      # remove valid transforms that didn't make it through this group
 451      poss_transforms = set([t for t, x in eq_under_transform.items() if x])
 452    # compile equivalences from all groups made under still valid transforms
 453    # a dict of sets so singletons aren't lost in finding connected components
 454    equivalents = {i: set() for i in range(self.n_tiles)}
 455    for group_equivalents in by_group_equivalent_tiles:
 456      for (tr, tile_i, tile_j) in group_equivalents:
 457        if tr in poss_transforms:
 458          equivalents[tile_i].add(tile_j)
 459    self.tile_matching_transforms = {
 460      k: v for k, v in self.tile_matching_transforms.items() if k in 
 461      poss_transforms}
 462    self.tile_transitivity_classes = []
 463    equivalents = nx.connected_components(nx.from_dict_of_lists(equivalents))

 465      transitivity_class = []
 466      for tile in self.tiles:
 467        if tile.base_ID in base_IDs:
 468          transitivity_class.append(tile.ID)
 469          tile.transitivity_class = c
 470      self.tile_transitivity_classes.append(transitivity_class)

 472  def get_potential_symmetries(self) -> dict[int, tuple[float]]:
 473    """Assembles potential symmetries of the tiling from symmetries of the 
 474    tileable.prototile and of the tileable.tiles. Removes any duplicates that
 475    result. Result is assigned to the tile_matching_transforms attribute.
 476    
 477    TODO: consider retaining the Symmetry objects as these carry additional 
 478    information that might facilitate labelling under a limited number of the
 479    symmetries not all of them.
 480
 481    Returns:
 482      dict[int, tuple[float]]: dictionary of the symmetries (transforms 
 483        actually) in shapely affine transform 6-tuple format.
 484    """
 485    self.tile_matching_transforms = {}
 486    n_symmetries = 0
 487    pt = self.tileable.prototile.geometry[0]
 488    for tr in Shape_Matcher(pt).get_polygon_matches(pt):
 489      if not tr.transform_type in ["identity", "translation"]:
 490        self.tile_matching_transforms[n_symmetries] = tr
 491        n_symmetries = n_symmetries + 1
 492    for tile in self.tiles[:self.n_tiles]:
 493      for tr in Shape_Matcher(tile.shape).get_polygon_matches(tile.shape):
 494        if not tr.transform_type in ["identity", "translation"]:
 495          self.tile_matching_transforms[n_symmetries] = tr
 496          n_symmetries = n_symmetries + 1
 497    for tile in self.tiles[:self.n_tiles]:
 498      sm = Shape_Matcher(tile.shape)
 499      transforms = [sm.get_polygon_matches(self.tiles[i].shape) 
 500        for i in self.shape_groups[tile.shape_group] if i < self.n_tiles]
 501      for tr in itertools.chain(*transforms):
 502        if not tr.transform_type in ["identity", "translation"]:
 503          self.tile_matching_transforms[n_symmetries] = tr
 504          n_symmetries = n_symmetries + 1
 505    self.tile_matching_transforms = self._remove_duplicate_symmetries(
 506      self.tile_matching_transforms)
 507    return self.tile_matching_transforms

 509  def _remove_duplicate_symmetries(self, transforms:dict[int,Any]):
 510    """Filters the supplied list of shapely affine transforms so that no 
 511    duplicates are retained.
 512    """
 513    uniques = {}
 514    for k, v in transforms.items():
 515      already_exists = False
 516      for i, u in uniques.items():
 517        already_exists = np.allclose(
 518          v.transform, u.transform, atol = 1e-4, rtol = 1e-4)
 519        if already_exists:
 520          break
 521      if not already_exists:
 522        uniques[k] = v
 523    return uniques

 525  def _find_vertex_transitivity_classes(self):
 526    """Finds vertex transitivity classes by checking which vertices align
 527    with which others under transforms in the tile_matching_transforms 
 528    attribute. The process need only determine the classes for vertices

 530    matched base_ID.

 592  def _match_geoms_under_transform(
 593      self, geom1:Union[Tile,Vertex,Edge], 
 594      geoms2:list[Union[Tile,Vertex,Edge]], transform:tuple[float]) -> int:
 595    """Determines if there is a geometry in the supplied patch onto which the
 596    supplied geometry is mapped by the supplied symmetry.
 597
 598    Args:
 599      tile (Union[Tile,Vertex,Edge]): element whose geometry we want to match.
 600      patch (list[Union[Tile,Vertex,Edge]]): set of elements among which a 
 601        match is sought.
 602      transform (tuple[float]): shapely affine transform 6-tuple to apply.
 603
 604    Returns:
 605      Union[int,tuple[int]]: ID of the element in patch that matches the geom1
 606        element under the transform if one exists, otherwise returns -1.
 607    """
 608    match_id = -1
 609    for geom2 in geoms2:
 610      if isinstance(geom1, Tile):
 611        # an area of intersection based test
 612        match = tiling_utils.geometry_matches(
 613          affine.affine_transform(geom1.shape, transform), geom2.shape)
 614      elif isinstance(geom1, Vertex):
 615        # distance test
 616        match = affine.affine_transform(geom1.point, transform).distance(
 617          geom2.point) <= 10 * tiling_utils.RESOLUTION
 618      else: # must be an Edge
 619        # since edges _should not_ intersect this test should work in
 620        # lieu of a more complete point by point comparison
 621        c1 = geom1.get_geometry().centroid
 622        c2 = geom2.get_geometry().centroid
 623        match = affine.affine_transform(c1, transform) \
 624          .distance(c2) <= 10 *tiling_utils.RESOLUTION
 625      if match:
 626        return geom2.base_ID
 627    return match_id

 629  def _get_exclusive_supersets(
 630      self, sets:list[Iterable[Any]]) -> list[Iterable[Any]]:
 631    """For the supplied list of lists, which may share elements, i.e. they are
 632    non-exclusives sets, returns a list of lists which are exclusive: each 
 633    element will only appear in one of the lists in the returned list. Uses
 634    networkx connected components function to achieve this based on a graph 
 635    where an intersection between two sets is an edge.
 636
 637    Args:
 638        sets (list[Iterable[Any]]): list of lists of possibly overlapping sets.
 639
 640    Returns:
 641      list[Iterable[Any]]: list of lists that include all the original 
 642        elements without overlaps.
 643    """
 644    overlaps = []
 645    for i, si in enumerate(sets):
 646      s1 = set(si)
 647      for j, sj in enumerate(sets):
 648        s2 = set(sj)
 649        if len(s1 & s2) > 0:
 650          overlaps.append((i, j))
 651    G = nx.from_edgelist(overlaps)
 652    result = []
 653    for component in nx.connected_components(G):
 654      s = set()
 655      for i in component:
 656        s = s.union(sets[i])
 657      result.append(tuple(s))
 658    return result

 835  def transform_geometry(self, new_topology:bool, apply_to_tiles:bool,
 836                         selector:str, type:str, **kwargs) -> "Topology":
 837    r"""Applies a specified transformation of elements in the Topology whose 
 838    labels match the selector parameter, optionally applying the transform to 
 839    update tile and optionally returning a new Topology object (or applying it 
 840    to this one). 
 841    
 842    Implemented in this way so that transformations can be applied one at a time
 843    without creating an intermediate set of new tiles, which may be invalid and
 844    fail. So, if you wish to apply (say) 3 transforms and generate a new 
 845    Topology leaving the existing one intact:
 846    
 847        new_topo = old_topo.transform_geometry(True,  False, "a", ...) \
 848                           .transform_geometry(False, False, "B", ...) \
 849                           .transform_geometry(False, True,  "C", ...)
 850    
 851    The first transform requests a new Topology, subsequent steps do not, and it
 852    is only the last step which attempts to create the new tile polygons.
 853    
 854    **kwargs supply named parameters for the requested transformation.
 855
 856    Args:
 857      new_topology (bool): if True returns a new Topology object, else returns 
 858        the current Topology modified.
 859      apply_to_tiles (bool): if True attempts to create new Tiles after the 
 860        transformation has been applied. Usually set to False, unless the last
 861        transformation in a pipeline, to avoid problems of topologically invalid
 862        tiles at intermediate steps.
 863      selector (str): label of elements to which to apply the transformation.   
 864        Note that all letters in the supplied string are checked, so you can 
 865        use e.g. "abc" to apply a transformation to edges labelled "a", "b" or 
 866        "c", or "AB" for vertices labelled "A" or "B".
 867      type (str): name of the type of transformation requested. Currently
 868        supported are `zigzag_edge`, `rotate_edge`, `push_vertex`, and 
 869        `nudge_vertex`. Keyword arguments for each are documented in the 
 870        corresponding methods.
 871
 872    Returns:
 873      Topology: if new_topology is True a new Topology based on this one with
 874        after transformation, if False this Topology is returned after the
 875        transformation.
 876    """
 877    print(
 878f"""CAUTION: new Topology will probably not be correctly labelled. To build a 
 879correct Topology, extract the tileable attribute and rebuild Topology from that.
 880""")
 881    topo = pickle.loads(pickle.dumps(self)) if new_topology else self
 882    transform_args = topo.get_kwargs(getattr(topo, type), **kwargs)
 883    if type == "zigzag_edge":
 884      for e in topo.edges.values():
 885        if e.label in selector:
 886          topo.zigzag_edge(e, **transform_args)
 887    # ---------------------------
 888    elif type == "rotate_edge":
 889      for e in topo.edges.values():
 890        if e.label in selector:
 891          topo.rotate_edge(e, **transform_args)
 892    # ---------------------------
 893    elif type == "push_vertex":
 894      pushes = {}
 895      for v in topo.vertices_in_tiles(topo.tiles[:topo.n_tiles]):
 896        if v.label in selector:
 897          pushes[v.base_ID] = topo.push_vertex(v, **transform_args)
 898      for base_ID, (dx, dy) in pushes.items():
 899        for v in [v for v in topo.points.values() if v.base_ID == base_ID]:
 900          v.point = affine.translate(v.point, dx, dy)
 901    # ---------------------------
 902    elif type == "nudge_vertex":
 903       for v in topo.points.values():
 904         if v.label in selector:
 905           topo.nudge_vertex(v, **transform_args)
 906    if apply_to_tiles:
 907      for t in topo.tiles:
 908        t.set_corners_from_edges()
 909    topo.tileable.tiles.geometry = gpd.GeoSeries(
 910      [topo.tiles[i].shape for i in range(topo.n_tiles)])
 911    topo.tileable.setup_regularised_prototile_from_tiles()
 912    return topo

 914  def zigzag_edge(self, edge:Edge, start:str = "A",
 915                  n:int = 2, h:float = 0.5, smoothness:int = 0):
 916    """Applies a zigzag transformation to the supplied Edge. Currently this will
 917    only work correctly if h is even.
 918    
 919    TODO: make it possible for odd numbers of 'peaks' to work (this may require
 920    allowing bidirectional Edges, i.e. storing Edges in both directions so that
 921    all Tile edges are drawn CW). The `start` parameter is a temporary hack for 
 922    this
 923
 924    Args:
 925      edge (Edge): Edge to transform
 926      n (int, optional): number of zigs and zags in the edge. Defaults to 2.
 927      start (str, optional): label at one end of edge which is used to determine
 928        the sense of h, enabling C-curves with an odd number n of zigs and zags 
 929        to be applied. Defaults to 'A'.
 930      h (float, optional): width of the zig zags relative to edge length. 
 931        Defaults to 0.5.
 932      smoothness (int, optional): spline smoothness. 0 gives a zig zag proper,
 933        higher values will produce a sinusoid. Defaults to 0.
 934    """
 935    v0, v1 = edge.vertices[0], edge.vertices[1]
 936    if n % 2 == 1 and v0.label != start:
 937      h = -h
 938    ls = tiling_utils.zigzag_between_points(v0.point, v1.point, 
 939                                            n, h, smoothness)
 940    # remove current corners
 941    self.points = {k: v for k, v in self.points.items()
 942                   if not k in edge.get_corner_IDs()[1:-1]}
 943    # add the new ones
 944    new_corners = [self.add_vertex(geom.Point(xy)) for xy in ls.coords]
 945    edge.corners = edge.vertices[:1] + new_corners + edge.vertices[-1:]
 946    if not edge.right_tile is None:
 947      edge.right_tile.set_corners_from_edges(False)
 948    if not edge.left_tile is None:
 949      edge.left_tile.set_corners_from_edges(False)

 962  def push_vertex(self, vertex:Vertex, push_d:float) -> tuple[float]:
 963    neighbours = [self.points[v] for v in vertex.neighbours]
 964    dists = [vertex.point.distance(v.point) for v in neighbours]
 965    x, y = vertex.point.x, vertex.point.y
 966    unit_vectors = [((x - v.point.x) / d, (y - v.point.y) / d)
 967                    for v, d in zip(neighbours, dists)]
 968    return  (push_d * sum([xy[0] for xy in unit_vectors]),
 969             push_d * sum([xy[1] for xy in unit_vectors]))

Implemented in this way so that transformations can be applied one at a time without creating an intermediate set of new tiles, which may be invalid and fail. So, if you wish to apply (say) 3 transforms and generate a new Topology leaving the existing one intact:

new_topo = old_topo.transform_geometry(True,  False, "a", ...) \
                   .transform_geometry(False, False, "B", ...) \
                   .transform_geometry(False, True,  "C", ...)

The first transform requests a new Topology, subsequent steps do not, and it is only the last step which attempts to create the new tile polygons.

**kwargs supply named parameters for the requested transformation.

Args: new_topology (bool): if True returns a new Topology object, else returns the current Topology modified. apply_to_tiles (bool): if True attempts to create new Tiles after the transformation has been applied. Usually set to False, unless the last transformation in a pipeline, to avoid problems of topologically invalid tiles at intermediate steps. selector (str): label of elements to which to apply the transformation.
Note that all letters in the supplied string are checked, so you can use e.g. "abc" to apply a transformation to edges labelled "a", "b" or "c", or "AB" for vertices labelled "A" or "B". type (str): name of the type of transformation requested. Currently supported are zigzag_edge, rotate_edge, push_vertex, and nudge_vertex. Keyword arguments for each are documented in the corresponding methods.

Returns: Topology: if new_topology is True a new Topology based on this one with after transformation, if False this Topology is returned after the transformation.
